# Create Canton Party

Allocate a Canton Network party ID for the authenticated user.

<Note>
  Canton integration requires enterprise coordination. Contact support for setup.
</Note>

## Authorization

<ParamField header="Authorization" type="string" required>
  Bearer token from login or OAuth2 flow.
</ParamField>

## Response

<ResponseField name="canton" type="object">
  Created Canton identity.
  <Expandable title="properties">
    <ResponseField name="partyId" type="string">
      Canton party identifier.
    </ResponseField>
    <ResponseField name="participant" type="string">
      Participant node hosting the party.
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.signum.id/accounts/canton \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```typescript TypeScript
const response = await fetch('https://api.signum.id/accounts/canton', {
  method: 'POST',
  headers: { Authorization: `Bearer ${accessToken}` },
});

const { canton } = await response.json();
console.log('Party ID:', canton.partyId);
```
</RequestExample>

<ResponseExample>
```json 201 Created
{
  "canton": {
    "partyId": "party::signum::user-abc123",
    "participant": "participant::signum",
    "createdAt": "2026-01-20T12:00:00Z"
  }
}
```

```json 409 Conflict
{
  "error": "Canton party already exists for this user"
}
```

```json 503 Service Unavailable
{
  "error": "Canton network unavailable"
}
```
</ResponseExample>
