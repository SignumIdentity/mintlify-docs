# Refresh Token

Exchange a refresh token for new access and refresh tokens.

<Note>
  Refresh tokens are rotated on each use. Always store the new refresh token from the response.
</Note>

## Request

<ParamField body="refresh_token" type="string" required>
  The refresh token from a previous login or refresh response.
</ParamField>

## Response

<ResponseField name="access_token" type="string">
  New JWT access token.
</ResponseField>

<ResponseField name="refresh_token" type="string">
  New refresh token. The old token is invalidated.
</ResponseField>

<ResponseField name="token_type" type="string">
  Always "Bearer".
</ResponseField>

<ResponseField name="expires_in" type="number">
  Access token lifetime in seconds.
</ResponseField>

<ResponseField name="user" type="object">
  Current user profile.
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.signum.id/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "rt_abc123def456..."
  }'
```

```typescript TypeScript
const response = await fetch('https://api.signum.id/auth/refresh', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    refresh_token: storedRefreshToken,
  }),
});

const { access_token, refresh_token } = await response.json();
// Update stored refresh token
localStorage.setItem('refresh_token', refresh_token);
```
</RequestExample>
